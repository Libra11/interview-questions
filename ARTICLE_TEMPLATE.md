# 面试题文章模板规范

## 文章结构要求

### 1. 文件头部 (Front Matter)

```yaml
---
title: 问题标题（简洁明了，直接提问）
category: JavaScript/Vue/React/CSS/HTML/算法/网络等
difficulty: 入门/中级/高级
updatedAt: YYYY-MM-DD
summary: >-
  简洁的文章摘要，2-3行描述文章核心内容和学习价值
tags:
  - 标签1
  - 标签2
  - 标签3
  - 标签4
estimatedTime: XX 分钟
keywords:
  - 关键词1
  - 关键词2
  - 关键词3
highlight: 一句话总结文章的核心价值和重点
order: 数字（用于排序）
---
```

### 2. 文章正文结构

#### 多问题形式（推荐）

- 采用多个递进式问题的结构
- 每个问题用 `## 问题 X：具体问题标题` 格式
- 问题之间用 `---` 分隔线分开

#### 问题设计原则

1. **从基础到深入**：第一个问题通常是基本概念
2. **逻辑递进**：每个问题都在前一个问题的基础上深入
3. **实用性强**：每个问题都应该是面试中可能遇到的

尽量少出现以下问题：

- 在实际应用中有什么优势/作用？
- 有哪些应用场景/最佳实践？
- 与其他技术的对比/区别是什么？

### 3. 内容编写规范

#### 文字要求

- **重原理轻代码**：更多篇幅用于解释原理和概念
- **通俗易懂**：使用简单明了的语言，避免过于技术化的表述
- **逻辑清晰**：每个段落都有明确的主题，段落间逻辑连贯
- **性能问题**： 尽量少说性能问题（不要单独一个块讲），除非是面试中必须的

#### 代码要求

- **精简为主**：代码示例要简洁，只展示核心概念
- **不求完整**：代码片段只需说明问题即可，不需要完整可运行
- **注释清晰**：关键代码要有注释说明，尽量多的添加注释

#### 代码示例格式

```javascript
// 简洁的代码示例
function example() {
  // 关键逻辑注释
  return result;
}

// 对比示例（如果需要）
// ❌ 错误做法
// ✅ 正确做法
```

## 写作风格指南

### 语言风格

- **口语化但专业**：既要通俗易懂，又要保持技术准确性
- **避免冗长**：句子简洁明了，避免复杂的从句
- **多用比喻**：用生活中的例子来解释抽象概念
- **结构化表达**：多使用列表、分点来组织信息

### 技术解释原则

1. **先说是什么**：明确定义概念
2. **再说为什么**：解释存在的原因和价值
3. **然后说怎么用**：展示基本用法
4. **最后说注意什么**：提醒关键点和最佳实践

### 避免的内容

- **过长的代码块**：单个代码块不超过 20 行
- **复杂的实现细节**：不需要展示完整的库实现
- **过多的配置示例**：重点关注核心概念
- **重复的内容**：避免在不同问题中重复相同信息

## 预估时间指南

- **初级文章**：15-20 分钟阅读时间
- **中级文章**：20-30 分钟阅读时间
- **高级文章**：25-35 分钟阅读时间

根据内容复杂度和深度来设定合理的预估阅读时间。
